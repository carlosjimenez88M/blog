De antemano quiero resaltar una verdad sobre el articulo

As programming languages go, there’s no denying that Python is hot

Python al ser diseñado y desarrollado por ingenieros tiene una versatilidad sobre la implementación de los modelos a producción más robusta a nivel de experiencia por parte de las plataformas como : AWS, SPark o Google CLoud ML.

A nivel de literatura por ejemplo las implementaciónes bajo BigQuery son más sencillas para Python que R y el soporte ni se diga, más ….. , esto no quiere decir que desde R sea extraordinariamente difícil. Simplemente que cada uno tiene una forma de integración según su público de uso. Hay que recordar que R tiene una orientación mayor a Científicos de datos de origen en ciencias : matemáticas, estadística y economía, mientras que Python está orientado a ingenieros (informaticos, de sistemas , etc).

Ahora sobre lo anterior hay que tener presente un aspecto fundamental, independientemente a cual usar, cada uno tiene sus particularidades. Es indudable que para hacer análitica de datos y modelos de Deep Learning R cuenta con una experiencia muy interesante en comparación a Python, dado el desarrollo del pensamiento estadístico que el mismo software poseé. Más Python gano la batalla en términos de escalabilidad , lo cual es una deuda a nivel de literatura y cursos por parte de R.
